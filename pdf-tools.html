Tarumak Studio's PDF tools are built on pdf-lib and PDF.js — open source libraries that run entirely in the browser. Every operation from merging to password-protecting happens locally on your device without any file being sent to a server.

The 15 tools cover every common PDF workflow: combining multiple PDF files into one with drag-and-drop page ordering, splitting a PDF into separate pages or custom page ranges, reducing file size with an adjustable quality slider, converting PDF pages to JPG images at 150 or 300 DPI, converting JPG and PNG images into PDF documents, adding 256-bit AES password encryption, removing passwords from PDFs you have the right to edit, rotating individual pages, removing unwanted pages, and rearranging pages in a visual organiser.

Unlike tools such as iLovePDF or Smallpdf that cap free users at 2 operations per day, Tarumak Studio imposes no daily limits.

Frequently Asked Questions

Is my PDF sent to a server?

No. All PDF processing uses pdf-lib and PDF.js running inside your browser. Your file is read using the browser's FileReader API and processed entirely in memory on your device. Nothing is transmitted to any server, which makes it safe for confidential documents including contracts, financial statements, and legal filings.

What is the maximum PDF file size?

There is no server-imposed limit. Practical limits depend on your browser's available memory. PDFs up to 200MB process reliably in most modern browsers. Very large PDFs with many high-resolution images may be slow — consider compressing the images first if you encounter performance issues.

Can I merge more than two PDFs at once?

Yes. The PDF Merger accepts unlimited files. Drag them into any order before merging and the output is a single PDF with all pages combined in the sequence you specified.

Does password protection use real encryption?

Yes. The PDF Password Protector uses 256-bit AES encryption — the same standard used by banks and government systems. Without the correct password, AES-256 encrypted PDFs are computationally infeasible to open by brute force.

Why is my compressed PDF still large after compression?

PDF compression primarily reduces image quality inside the document. PDFs that contain mostly text with no images will not reduce significantly because text is already stored very efficiently. For image-heavy PDFs, set the quality slider to 60–70% for substantial size reduction with acceptable visual quality.
